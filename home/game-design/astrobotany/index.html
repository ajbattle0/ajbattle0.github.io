#2 Organisation and planning

To manage this project we used a series of software:

Similarly to how we worked on the previous game jam for this game jam we began by using Miro to get ideas from the team down. We wrote down any ideas we had for the game whilst actively discussing ideas over voice chat.

The advantages of this process improves the creative process in a few ways:

  1. Ideas are written down and so aren’t forgotten.
  2. Provides a visual representation of what everyone is thinking during the discussion so makes it easier to understand other peoples perspectives.
  3. Saves time getting ideas out as people don’t need to wait for a chance to speak to talk about their ideas.
  4. Allows people to collaborate with other peoples ideas.
  5. Keeps ideas small and understandable by sticking to the post it note size.
  6. Allows less confident speakers to collaborate.

Using this method we were able to come to a quick consensus as to what we wanted for the game:

  • A older lesbian main character on a space ship who has to grow a plant.
  • Looking after a plant through three stages of growth.
  • A futuristic feel to the spaceship and a alien feel to the plant.
  • The plant requiring three types of nourishment, water, meat and music (because its alien)
  • A cinematic at the start and the end of the game.

Secondly once we had the games idea we broke this down into the different tasks that would be required. We wrote these out on trello. Which would allow the team to keep a record of everything that was being worked on and by who. This also made it easier to visually see everything that needed to be done.

Finally we used discord for chatting and sharing progress. I often used it for the one to ones I had with the team to make sure that everyone was doing well and helping connect the art of the game with the programming required.

We began by assigning roles:

  • Myself: Create the art for the plant and animate it.
  • Kit Westall (twitter.com/kibblemeat); create the art for the game and the cinematics.
  • Kerrie Holland (https://www.linkedin.com/in/hikerrie/); Find audio for the game.

#3 UI

I set up a simple UI using the same system that I used within the Death is only the beginning game.

This was done simply with this method:

  • Create a canvas that the menu is on with buttons to the different sub menus.
  • Make a script that has a list of public voids that opens scenes.
  • In the on click() part of the button add the object the script is attached to and then choose the public void that changes to the required scene.
  • If the button is opening a sub menu then instead the canvas that the menu is attached to is enabled and the menu that is being left is disabled.

I created a few assets for the menu. The buttons and the title:

For the buttons I wanted to reflect the spaceship background that Kit had created and create something that still was legible. As for the Title I went with two fonts that had a space feel to them adding planet rings to the O and a green gradient to botany to connect it to botany.

With video games you are able to be a lot more extravagant with logo design but I went with a more clean design to fit with the space theme and to make it easier for the team to share the project with clients who might not know as much about video games.

#4 Coding

Unfortunately our teams coder was unable to work on this project. Therefore I took over the position of programmer.

We decided to go with a Rhythm game gameplay as that was what I felt comfortable coding.

Our concept artist Kit quickly created an idea for how this would work that I used as a reference.

In this a token spawns in line with its corresponding colour. If the player hits the button corresponding to the colour they get a point. If they miss it they lose a penalty. If they lose too many penalties they lose and the level reset. Get enough and the game moves onto the next level.

Further down inside of void up. This part of the code creates the token. The way it works is there is a float called TokenInterval that with every update has +1 added to it. Then this reached whatever TokenIntervalAmount is set to. The TokenInterval is reset and the code inside in the if statement is played.

WhichTokenRandomNumber is set to a number in between 1 and whatever LevelDifficulty is set to. If set to 4 the random number can be a number that corresponds to creating either a red, blue or green token. At 6 it can also spawn a red and a blue or a green and a red. At 8 it can also spawn a blue and a green or all three at once.

This then invokes the public void from earlier corresponding to the tokens required to make.

Having TokenIntervalAmount and LevelDifficulty be floats that can be altered it makes it easy to alter how fast the tokens come down and how complex the input is.

No. 4 Winning or losing the game

Once again inside of the SpawnTheTokens script in void update is some code to check if the values of GameScore or PenaltyScore are high enough to either win or lose the level.

If they are a win/lost messages is activated and a timer is added either gamewon or gamelost. Once this gets to 8 the game checks what level the player is on and either moves onto the next level if the player won or repeats the level if the player lost.

#6 Art for Plants

It was also my job to create the plants for the game that the botanist is growing. I started this process by drawing the plants and then scanning them:

These three trees representing seedling, adult and flowering where then cropped out using the pen tool in affinity designer.

I then set the line drawing to linear burn and shaded the plant. Using mostly my favourite shading brushes.

Next I used gradient maps for each of the plants on top of the shading which allowed me to colour the plants easily.

I then painted on top of the plants adding extra details and colours.

These plants were then brought into unity and arranged alongside Kits artwork to create the scenes.
