I recently took part in the GameDev.tv 2022 Game Jam! This was a collaborative project between myself and a team of 8 others. Our theme was Death is only the beginning.

Here are a list of the different sections that I worked on:

  1. Research
  2. Organisation and planning
  3. UI and coding
  4. Dialogue and Yarn Spinner
  5. Audio
  6. Art for parallax
  7. Game thumbnail
  8. Launch and bug fixing
  9. Outcome
  10. Reception

#1 Research

We first found out about the game sometime before and everyone that was interested in working on the project expressed their interest within our game development group.

We planned a meeting for once the games theme had been decided. The game jam published a list of possible ideas for the game and as the unapologetic emo that I am I was happy to see that the Death is only the beginning theme has been chosen.

Alot of the research for this project had already been done earlier as I had become very familiar with the way that unity works and would be using a lot of software that I had a strong understanding of already.

#2 Organisation and planning

To manage this project we used a series of software:

Firstly the task was planned using miro to visually represent what the teams ideas. We came up with as many words that we could associate with the games theme and then used those as prompts to brain storm the game theme.

Secondly once we had the games idea we broke this down into the different tasks that would be required. We wrote these out on trello. Which would allow the team to keep a record of everything that was being worked on and by who.

Thirdly we setup a github that would allow us to share the unity project between the team and make changes independently of each other.

Finally we used discord for chatting and sharing progress. I often used it for the one to ones I had with the team to make sure that everyone was doing well and helping connect the art of the game with the programming required.

#3 UI and Coding

After we had assigned some of the starting tasks, backgrounds, music and concepting I left myself open to fill the positions that the team required and act as the go between for the artists work and the games programming.

I started by setting up the games UI system that would be important across the game. To do this I created a canvas that would allow me to swap between the different scenes within the game.

This basically created a list of public voids that when called would change the level. I then assigned each of these to the corresponding buttons in the games UI’s

With this I created a simple set of menus with the following attributes:

Main menu

Goes to:

Options menu by closing Main menu and then opening a new canvas underneath it with the options title and a button to return to main menu by reversing the process.

Credits menu by closing Main menu and then opening a new canvas underneath with the credits title, the credits and a button to return to main menu by reversing the process.

Start game; this runs the Load_Introduction void which uses the scene manager to open the first level.

End game; this runs the QuitGame void which plays the code: Application.Quit which just quits the game.

In game menu

The same as the menu above it has a credits and a options menu but it doesnt have a quit game button instead it has a return to main menu and a back to game button that hides the menu.

With this completed the teams UI designer was able to build on top of this to create some good looking UIs for the game, I helped this also by providing a background and editing some of the UI elements in unity for them.

#4 Dialogue and Yarn Spinner

As this game was going to be dialogue heavy with a short development time it made sense to me to Yarn Spinner.

Yarn Spinner is a open source dialogue system for unity. The way that it works is you write dialogue for your game using the yarn spinner language in a yarn editor which is a very simple and intuitive way to write.

This is then uploaded into unity and put into your yarn project which handles all of the hard work for the game such as displaying text, displaying dialog options for the player that are dependant on their previous choices, scrolling text, click to continue the conversation and knowing where abouts the player is in the conversation.

With this the teams dialogue writer wrote the dialogue and then together with them I implemented the dialogue into the game and wrote the dialogue script that would handle the games dialogue.

This dialogue system was then further built upon by the teams main programmer who was able to use all of the public voids we had created to assign the dialogue to different parts of the game.

#5 Audio

The team was very lucky to get a composer who wanted to make music for the game and put together a lovely set of tracks for the game.

However the game still needed an ambient track to mix with the audio and help sell the scene that this boat is on a lonely river in the underworld.

After looking at a few options I used Ambient mixer to create this sound track. This mixer allows you to add different sounds into a mix and has them play randomly throughout to build an ambient soundscape.

I started with some water noises, added wind, thunder and rustling wind but didn’t add any animal noises to give the impression of a dead and empty place.

Next we also needed some noises on the buttons to make them feel interactive, this was achieved by writing a simple audio script, attaching it to the button and then creating the different events that the button would interact with. Those being click and hover over.

In the code I have a noise chosen for if the button is clicked that will play and one for if it is hovered over that will play.

#6 Art for parallax

For the boat journey we wanted to give the impression of movement. This would be achieved through parallax. A system where the scene is broken up into different assets which loop at different speeds to give the illusion of movement.

Here is an example of the early parallax that was created and used to inform the teams artists:

To help the art team I started by creating this simple vector concept for the scene and then sent it to them and to the programmer so they would be able to use it to code the parallax while the team drew the assets.

Personally I created the water and the grass layers.

These where made by drawing subsections of waves and grass and then combining them together whilst at each end putting a section that had been cut in half so it would match up with the other end when it looped.

I also made the sky by taking my original art and adding a paper texture to it.

When I got the background artists work I formatted that so it would work with the programmers code and fit the scene by shading it and cropping out the sky.

The background artists work combined with other elements created a very atmosphere feeling environment.

#7 Game thumbnail

For the games thumbnail I created a black and white drawing then shaded it by adding vector layers on top with different shades of grey set to multiply. I then added the text, added a white glow to make it feel more ghostly before added the same to the moon and the boats lantern.

#8 Launch and bug fixing

On the day of the game hand in we started a call at 4pm with the team and kept each other company as we worked through the issues with the game. However when we got to the end of the list of issues and built the game it wouldn’t load on itchio.

After some hard work I tried rolling back the unity version to a 2020 version which fixed the engines issues and allowed us to build the game.

#9 Outcome

With the game completed we uploaded it to itchio for the game jam and were able to meet the deadline and create a lovely game for people to play during the event.

#11 Lessons

One of the things that came out of this project is there were a few episodes of team difficulties and had to deal with some rudeness and unacceptable comments.

A few of these moments felt unfair on myself and they occurred in a way that would allow me to have been able to bite back and revel in my own self righteous fury knowing that I would have been perceived as being in the right.

However as this was another project where I was fulfilling the role of project manager understood that I was in a position of responsibility to team members to put their needs before my own.

By being kind and forgiving in this situation, by moving past the instinct to be angry something really beautiful emerges the relief of not causing another person harm and not causing a person extra distress on top of the distress that might be making them act out.
