Working on the game

This describes how I completed work for this project.

While recently working on a unity course I was able to chose a commercial project to work on.

We got to pick our commercial projects from a list of potential projects, reading through these briefs I found the brief for this project, return to the garden to be interesting and therefore applied.

Luckily I was accepted to this project. At the start I took up the role of project manager and began working with the client and the team to create the project.

The project was to create a mobile game set inside of a garden with humanoid animals living in their own little zone which was then related to the music period that they represented.

This was a very similar direction to that which we took with the previous project however now there were a few extra key features:

  1. There was no coder on the team we would we coding the game ourselves.
  2. The game was set in a garden rather then a game studio and so would need a world creating aswell including environments that the characters would live in.
  3. The game would have require a story.
  4. The game was for phone.
  5. The game was in 3D and would have an AR component.

Step #1 Research

After meeting with the client and discussing with the team we were able to find out what people were interested in working on and then assign roles accordingly.

One of the starting tasks was to research about the different aspects of the game. I wrote a number of reports regarding this information. Learning about the different systems.

The brief required that each character would have their own zone within a “garden” for this I looked for inspiration.

For this I looked at:

  1. Buildings that represent a time period well.
  2. Buildings that exist well within nature with an empathises on camping.
  3. Interesting environments for the buildings to exist in.

Using these references I came up with some concept art, ultimately another member of the team wanted to work on concept art so these further developed their concept art however we both seemed to be on a similar page in regards to art direction.

Step #2 Planning the project

After reading the brief and speaking to the client it was important to plan the project out.

We had two months to work on this project and so we had to make sure that we kept to a schedule.

We used a mindmap as a way to plan out all of the required tasks.

We also looked at a number of games and artists for inspiration, these we made into large pintrest boards:

Step #3 Leading the team

As the team leader it was important for me to manage the team and work towards a successful out come for the team. I achieved this through these methods:

  1. Making sure every member of the team could work on what they wanted to; I wrote out all of the tasks required for the game to be completed and let the team pick what they wanted to do from the list. I also asked the team at the start what areas they were interested in working on and assigned them to those roles.
  2. Creating a trello; I used a number of Trello boards to keep track of the project showing the team what people were working on aswell as what needed to be done so everyone was in the loop.
  3. I wrote agendas for meetings; this meant there was a understanding of what each meeting was for.
  4. I wrote minutes from each meeting and used these to create reports. This also helped members who missed meetings.
  5. I had a open line of dialogue with the team so they could talk to me whenever they wanted to.

I wrote in depth about the lessons I learnt from running this project here.

Step #4 Creating assets

For my parts of the project I was going to create a series of menus for the project that would allow the player to go between the different areas of the game.

Menu 1 : Main menu

I knew that I was very interested in using the unity engine to create a parallax effect and so took some assets I had created for another project and altered them to be appropriate for the project.

With the first test a success I created assets intentionally for the menu.

For inspiration I looked at photos of the Yorkshire Dales and drew them in a way where the different layers would be able to move past each other realistically.

These layers included:

  1. A sky layer; a simple gradient.
  2. A sun, a simple circle with a pink glow.
  3. Clouds; pink clouds on a transparent background.
  4. Mountains; drawn by hand and then coloured digitally.
  5. Hills.
  6. A wall and a van.
  7. Foreground grass and trees.
  8. Lavender sprigs.

Menu 2 : Map menu

This menu was to be for when you go to the different zone. As inspiration for this zone I looked back on the places I had been in the past that I felt were good examples of beauty and so was heavily inspired by Keswick.

I also worked with the teams concept artist and redrew their zones as small icons.

Colouring

To colour this menu I took the project into affinity designer and got to work.

  • To make use of the drawing I set the drawing to linear burn and put it on the top layer.
  • I added a shadow layer with the median colour for project (Note I didn’t really shade this project but this step would make it easier to do so in another project.) This was sent to the back of the layers.
  • I began separating out the different areas of the drawing such as tree as fields. I separated out the sky and water then drew colours into them whereas the layers with none reflective were set to colour.

To make the drawing alot more interesting I used gradients for most of the colour layers.

Step #5 Animating the assets

I took these assets into unity and created a simple animation for them. This was achieved by having each layer move a different distance based on its position in the scene.

The layers at the back would move the least whilst those in the front move further. This creates the illusion of depth know as parallax.

On top of this is I created a canvas in unity and added buttons and other menu elements. I then wrote code to have each of these elements lead to different locations and animated the buttons so that they felt more intuitive.

Step #6 Integration into the game

For each level of the game I created a pause menu and a score menu and used code to integrate these into the game the pause menu would allow you to return to the start of the game and the score menus counted the games points.

Step #7 Working with the team

At the start of this project we had some sessions to learn how to work together well in a team. From this I took a fair few points and made an effort to communicate well and postively.

Step #8 Final outcome

Once all of the different parts of the game were completed I put them together. In the final game

The Dialogue was written by a member of the team. I used yarn spinner to make their dialogue into a fully interactive dialogue tree where the player is able to pick different responses in the dialogue.

The main menu with the parallax I implemented. I also added a title for the game and a button that plays the cinematic one of the team had written.

The map menu with the different zones floating.

The 70s zone. I wrote code so that the player could rotate the environment and also animated the different elements the team had made.

The maze zone; for this zone I wrote code to make the player character move and collect gems when they collided with them.
